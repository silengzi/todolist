下面给你一份 **生产级、稳定、易维护的 Next.js 自托管部署方案（Git + Docker Compose + Nginx）**。
特点：

* 不使用 Docker Hub 存储你的镜像
* 使用 Git 更新代码
* 使用 Docker 本地构建运行
* 使用 Nginx 提供域名访问和反向代理
* 可长期稳定运行

---

# 一、最终架构图

```text
开发机
  │
  │ git push
  ▼
服务器
  │
  ├─ Git（拉取代码）
  │
  ├─ Docker Compose
  │    ├─ Next.js container (内部3000)
  │    └─ Nginx container (外部80)
  │
  ▼
用户访问
http://your-domain.com
```

数据流：

```text
浏览器 → Nginx → Next.js → 返回页面
```

# 五、Dockerfile

```dockerfile
FROM node:20-alpine

WORKDIR /app

COPY package.json package-lock.json ./

RUN npm ci

COPY . .

RUN npm run build

EXPOSE 3000

CMD ["node", ".next/standalone/server.js"]
```

说明：

* build Next.js
* 使用 standalone server.js 运行

# 六、Nginx 配置

创建：

nginx.conf

```nginx
server {

    listen 80;

    server_name your-domain.com;

    location / {

        proxy_pass http://nextjs:3000;

        proxy_http_version 1.1;

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;

        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';

    }

}
```

如果没有域名，可改为：

```nginx
server {

    listen 80;

    server_name _;

    location / {
        proxy_pass http://nextjs:3000;
    }

}
```

# 七、docker-compose.yml（核心）

```yaml
version: "3"

services:

  nextjs:
    build: .
    container_name: nextjs-app
    restart: always

  nginx:
    image: nginx:alpine
    container_name: nginx
    restart: always
    ports:
      - "80:80"
    volumes:
      - ./nginx.conf:/etc/nginx/conf.d/default.conf
    depends_on:
      - nextjs
```

说明：

```text
nextjs:
  内部运行 3000端口

nginx:
  外部暴露 80端口
  自动转发到 nextjs
```

# 八、服务器部署流程（完整步骤）

## 步骤 1 创建目录

服务器执行：

```bash
mkdir -p /app
cd /app
```

---

## 步骤 2 拉取项目

```bash
git clone https://你的git仓库 nextjs-app
```

进入目录：

```bash
cd nextjs-app
```

---

## 步骤 3 启动服务

```bash
docker compose up -d --build
```

说明：

```text
--build  自动构建镜像
-d       后台运行
```

---

## 步骤 4 查看运行状态

```bash
docker ps
```

应看到：

```text
nextjs-app
nginx
```

---

## 步骤 5 浏览器访问

```text
http://服务器IP
```

成功。
